MRI.COM実行環境 MXE

MRI.COM実行環境 (MRI.COM eXecution Environment; MXE)とは、MRI.COM実験の前処理、実行、後処理、解析のプログラムを集約したパッケージです。 MRI.COM利用者の共通基盤となることを目指しています。

概要

特徴

  • MRI.COMの前処理、後処理、解析ツールを集約
  • いくつかの設定込みパッケージのベースに使われている
  • 地形の読み込みなどのルーチンをライブラリとして提供
  • コミットルールの違いから、MRI.COMとは別パッケージとして管理
  • いくつかのツールは単体テストによって動作が確認されている
  • 動作実績_ 矩形海モデル、全球渦なしモデル GONDOLA_100、日本沿岸モデル、瀬戸内海モデル

内容

各ディレクトリの内容です。詳細は README-MXE.md を参照。

  • exp 実験の実行
  • prep 前処理
  • postp 後処理
  • anl 解析
  • setting 設定
  • lib MXE独自のFortranライブラリ
  • linkdir データディレクトリへのシンボリックリンク
  • grads/ gradsスクリプトを置きます。
  • doc/ 実験に関するドキュメントを置きます。

開発

  • 気象研究所 全球大気海洋研究部第4研究室
  • 連絡先窓口：坂本 (ksakamot@mri-jma.go.jp)

バーション

MRI.COMに合わせて月1回の更新。 更新内容 (GitHub MXEの登録が必要)

使い方

ライセンス

MRI.COMにならいます。(第三者への提供や一般公開は許可していません。)

必要要件

  • Fortranコンパイラ (MPIは使わない)
  • make

取得方法

  • 気象庁、気象研の方：本庁Redmineを参照
  • 貸与等利用の方：GitHub MXE (登録者のみ可)
  • その他の方：MRI.COM の貸与申請をお願いします

チュートリアル

  • 使用方法はパッケージ中の README-MXE.md を参照
  • MXEをベースに実験パッケージを作る
  • 実験パッケージのMXEをアップデートする
  • 相談窓口: GitHub - MXE

開発に参加するには

パッケージを拡充・修正する開発フローは以下のとおり。

気象研究所、気象庁の方

  1. 本庁Redmine への参加申請を坂本( ksakamot@mri-jma.go.jp )に行う
  2. 変更内容を4研メンバーに相談する
  3. Redmineでチケットを作成し、ウォッチャーに相談した人を追加する
  4. ソースコードが完成すれば、開発管理サーバ上のmasterを直接に更新し、チケットのステータスを「解決」とする。
  5. (その後、4研メンバーがmasterのコードをテストし、必要であれば勝手に修正します。テストが終わればチケットのステータスを「終了」にします。)

外部機関の方

  1. GitHub - MXE への参加申請を坂本( ksakamot@mri-jma.go.jp )に行う
  2. 変更内容を坂本に相談する
  3. GitHubでIssueを作成する
  4. コードが完成すればプル・リクエストを送る
  5. 坂本がコードをテストし、masterに取り込みます。必要であれば勝手に修正します。
