MRI.COM

MRI.COMは気象庁気象研究所で開発されている海洋大循環モデルです。

概要

モデルの特徴

  • 水平一般直交座標(Arakawa Bグリッド)、深度座標(z*または z-σ)、自由海面、静水圧、ブシネスク近似
  • 多数のオプション
  • 気象庁の現業運用で使用されている
  • 気候システムモデルの海洋パート、海洋データ同化システムのモデルパートとしても利用されている
  • CMIPに参加し、最先端の機能を維持
  • 2000年頃から開発が継続
  • その他の概要

開発

  • 開発チーム: 気象研究所海洋・地球化学研究部第一研究室
  • 連絡先：坂本 (ksakamot@mri-jma.go.jp)
  • ドキュメント　マニュアル、論文、発表資料など
  • ノート

バージョン

  • 開発版 MRICOMv4.5 (2017/10-)
  • 安定版 MRICOMv4.4 (2017/10)
  • 今後のロードマップ
  • リリース・ノート
  • 過去の安定版
    • MRICOMv4.2 (2016/09)

関連プロジェクト

  • MRI.COM実行環境「MXE」
  • 海洋データ同化システム MOVE
  • 気象研究所地球システムモデル MRI-ESM
  • 日本沿岸海況監視予測システム「MOVE/MRI.COM-JPN」 (気象庁で現業運用予定)

モデルを使うには

ライセンス

  • 気象庁海洋モデルの利用申請参照
  • 論文に使う場合は、以下のマニュアルを引用してください (可能ならTsujino et al. (2011) JOも加えて)。
    • Tsujino et al., Reference Manual for the Meteorological Research Institute Community Ocean Model version 4 (MRI.COMv4), 気象研究所技術報告, 80, 2017, DOI:10.11483/mritechrepo./80
    • 論文等においては「MRI.COMv4」、またはバージョンを特定する場合には 「MRI.COMv4.2」, 「MRI.COMv4.4.Beta02」 などと表記する。1)

必要要件

  • Fortran 90/95以降
  • MPIライブラリ (MPI並列の場合。なしでも実行可)
  • make
  • git (ソースコード入手に必要)

取得方法

  • 気象庁、気象研の方：本庁Redmineを参照(設定込みパッケージは下を参照)
  • モデル共同研究の方：JAMSTEC Redmineを参照
  • その他の方：気象庁海洋モデルの利用申請から申請をお願いします
    • ソースコードの取得

使い方

  • 設定済みパッケージを動かす
    • 矩形海モデル・パッケージ「rectangle」 初めての方はこれを動かしてみることを勧めます
    • 瀬戸内海モデル・パッケージ「MRI.COM-Seto」
    • 全球渦なしモデル・パッケージ「GONDOLA_100」
  • 設定から作成する
  • モデル実行中に問題が起こったとき

サポート

バグを見つけたなどサポートが必要であれば以下に書き込んでください。

  • 気象庁、気象研の方：本庁Redmine - フォーラム

動作実績

様々なスパコン、ワークステーション、PCで動作する。詳細は矩形海パッケージを参照。

開発に参加するには

  • 気象研究所、気象庁の方： 本庁Redmine に参加してください
    • 開発ではMRI.COM 開発者向けの情報(内部ページ)を参考にしてください
  • その他の方：github (準備中)でpull request してください
